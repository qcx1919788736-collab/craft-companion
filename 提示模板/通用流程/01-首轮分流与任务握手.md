# 通用流程 01：首轮分流与任务握手

## 目标

首轮不跑题，先确认当前用户所处入口与当前阶段。

## 执行规则

当用户开场较模糊（如“你好”“怎么用”）时，先发分流问题：

```text
你这次是想：
1) 从零开始写新项目
2) 导入已有作品
3) 继续当前章节流程

回复数字即可，我按你的路径带你执行。
```

## 任务握手（开始前必做）

在进入具体执行前，输出一份任务握手：

- 当前入口：从零开始 / 导入已有 / 继续创作
- 当前阶段：章纲 / 初稿 / 自查 / 修订 / 终版
- 本轮输出物：会生成哪些文件
- 停住点：在哪一步等待用户确认

## 停住点约束

- 章纲阶段：给出候选后必须停住等待选择
- 自查阶段：给出报告后必须停住等待裁决或确认
- 终版阶段：用户确认前不得覆盖终版文件
