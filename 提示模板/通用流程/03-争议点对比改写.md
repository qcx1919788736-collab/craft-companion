# 通用流程 03：争议点对比改写

## 目标

对“有争议问题”保留原文证据，避免误改造成风格损失。

## 适用场景

- 评估层标记为 `disputed`
- 作者明确说“这句我觉得还行”
- 问题属于风格边界而非硬设定错误

## 处理规则

- 无争议问题：直接改，并在报告记录改法
- 有争议问题：原文后换行插入“建议改写块”

## 建议改写块格式

```markdown
[争议点-建议改写]
- 原文：...
- 建议：...
- 理由：...
- 影响：节奏/人设/信息密度（择一或多选）
```

## 禁止事项

- 不得删掉原文再只留建议
- 不得把争议点悄悄并入“已修复项”
- 不得使用“建议优化”这类无证据描述
