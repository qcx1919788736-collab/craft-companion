# 引导Prompt 03：设计故事结构

## 使用场景

你已经有了核心概念和主角设定，现在需要规划故事的整体结构。

## 前置条件

已完成 `01-定义核心概念.md` 和 `02-构建主角.md`。

## 任务目标

帮助作者规划故事的大致走向和节奏，生成初始的章节规划和时间线框架。

## 引导流程

### 第一步：确定故事起点

**AI提问**：

1. 故事从什么事件开始？（打破日常的触发事件）
2. 这个事件发生在什么时间、什么地点？
3. 主角在事件发生前的日常是什么样的？

**记录**：触发事件、时间地点、日常状态

### 第二步：确定故事终点

**AI提问**：

1. 你心目中的结局是什么？（可以模糊）
2. 主角最终的状态是什么？
3. 核心冲突如何解决？（完全解决/部分解决/开放式）

**如果作者不确定结局**：
- 这完全正常，可以先定一个方向
- 提供2-3种可能的结局类型供参考
- 标注"暂定，可随创作调整"

**记录**：结局方向、主角终态、冲突解决方式

### 第三步：关键转折点

**AI提问**：

在起点和终点之间，你能想到哪些关键转折？

引导思考：
1. 主角第一次面临真正的考验是什么？
2. 故事中最大的反转/揭示是什么？
3. 主角最低谷的时刻是什么？
4. 主角做出最关键决定的时刻是什么？

**不需要全部回答**，有几个就记几个。

**记录**：关键转折点列表

### 第四步：节奏规划

**AI提问**：

1. 你希望故事节奏偏快还是偏慢？
2. 紧张段和舒缓段的比例大概是多少？
3. 有没有你特别想写的场景类型？（战斗/日常/情感/悬疑揭示）

**AI建议**：

根据作者的回答，提供节奏建议：
- 长篇建议每3-5章有一个小高潮
- 每10-15章有一个大转折
- 紧张段和舒缓段交替，避免连续多章同一基调

**记录**：节奏偏好、场景偏好

### 第五步：章节粗略规划

**AI提问**：

基于以上信息，我们来粗略规划前10章：

1. 第1-3章：建立日常 + 触发事件
2. 第4-6章：主角应对变化
3. 第7-10章：第一个小高潮

对每个阶段：
- 大致发生什么？
- 涉及哪些人物？
- 情绪基调是什么？

**注意**：这只是粗略框架，不是详细大纲。后续创作中可以随时调整。

**记录**：前10章粗略规划

### 第六步：确认与输出

**AI总结**：

生成两份文档：

1. **时间线框架**（`知识库/03_故事进展/时间线_完整版.md`）
   - 故事时间跨度
   - 关键时间节点
   - 标注"框架，待创作中细化"

2. **章节规划**（`工作区/章纲.txt`）
   - 前10章粗略规划
   - 后续章节方向提示
   - 标注"初始规划，可随时调整"

## 输出

- `知识库/03_故事进展/时间线_完整版.md`
- `工作区/章纲.txt`

## 注意事项

### 不要过度规划

- 前10章可以稍微详细，后面越来越粗略
- 故事会在创作中自然生长，过度规划反而限制创作
- 关键转折点定下来就够了，中间的过渡留给创作时发挥

### 保持灵活性

- 所有规划都标注"可调整"
- 不要让作者觉得必须严格遵循规划
- 规划是指南针，不是铁轨

### 关注节奏而非情节

- 比起"第5章发生什么"，更重要的是"第5章的情绪基调是什么"
- 节奏规划比情节规划更有价值
- 确保紧张和舒缓交替

## 示例输出

```markdown
# 章节规划（初始框架）

> 这是初始规划，创作过程中可随时调整。

## 第一阶段：日常与裂缝（第1-3章）

### 第1章
- **事件**：李明的普通大学生活，某天晚上发生异常事件
- **情绪**：平静 → 不安
- **涉及人物**：李明、室友

### 第2章
- **事件**：李明试图忽略异常，但异常再次出现
- **情绪**：压抑、否认
- **涉及人物**：李明

### 第3章
- **事件**：异常无法忽视，李明被迫面对
- **情绪**：恐惧 → 接受
- **涉及人物**：李明、[关键人物]

## 第二阶段：应对与探索（第4-6章）
...

## 后续方向提示
- 第一个大转折预计在第10章左右
- 中期目标：[模糊描述]
- 结局方向：[模糊描述]
```

---

**使用方式**：在完成核心概念和主角设定后，将此prompt发送给AI继续引导。
