# 导入Prompt 05：识别伏笔线索

## 任务目标

从用户提供的小说文本中，识别所有伏笔、线索、悬念和未解之谜，生成符合 `00_核心上下文/伏笔追踪表_完整版.md` 格式的伏笔追踪表。

## 输入

用户提供的小说文本（已完成的章节）

## 输出格式

遵循 `template/知识库/00_核心上下文/伏笔追踪表_完整版.md` 格式

## 识别规则

### 1. 什么算伏笔

以下类型的内容应被识别为伏笔：

- **未解释的异常**：文本中出现但未解释的奇怪现象/事件
- **角色隐瞒**：角色明显知道但没有说出的信息
- **暗示性细节**：看似随意但被特意描写的细节
- **未完成的事件**：开始但未结束的事件线
- **角色预言/暗示**：角色说的意味深长的话
- **信息差**：不同角色掌握的不同信息
- **反常行为**：角色做出不符合常规的行为
- **物品/符号**：被特意提及的物品或反复出现的符号
- **未接的电话/未读的信**：被延迟的信息传递

### 2. 伏笔分类

- **角色伏笔**：关于角色身份/过去/秘密的线索
- **情节伏笔**：关于未来事件的暗示
- **设定伏笔**：关于世界观规则的线索
- **关系伏笔**：关于人物关系变化的暗示
- **悬念**：明确提出但未回答的问题

### 3. 判断伏笔状态

- **已埋**：只出现了一次，尚未推进
- **推进中**：在后续章节中有进一步展开
- **待收束**：已充分铺垫，读者期待揭示
- **已收束**：已在文本中完成闭合
- **疑似伏笔**：不确定是否是有意为之的伏笔

### 4. 记录植入方式

每个伏笔标注其植入方式：
- **体感**：通过角色的身体感受植入
- **对话**：通过角色对话带出
- **双线信息差**：不同视角/线索之间的信息不对称
- **日常动作**：嵌入日常行为中
- **环境细节**：通过环境描写暗示

## 处理边界情况

### 不确定是否是伏笔

- 标注为"疑似伏笔"
- 列出判断依据
- 建议作者确认

### 伏笔密度过高

- 优先记录明确的伏笔
- 疑似伏笔单独列出
- 不要为了数量而过度解读

### 已收束的伏笔

- 仍然记录，标注为"已收束"
- 记录埋设和收束的章节
- 有助于理解作者的伏笔风格

## 输出示例

```markdown
# 伏笔追踪表

## 活跃伏笔

### 神秘包裹
- **状态**：推进中
- **埋设章节**：第1章
- **描述**：张三收到一个没有寄件人的包裹，里面是一把旧钥匙
- **植入方式**：日常动作（收快递）
- **推进记录**：
  - 第1章：收到包裹，钥匙放在抽屉里
  - 第4章：张三发现钥匙上有奇怪的刻痕
- **预计收束**：未知
- **收束方向**：钥匙可能对应某个锁/地点

### 李四的秘密
- **状态**：已埋
- **埋设章节**：第2章
- **描述**：李四深夜偷偷打电话，被张三撞见后慌张挂断
- **植入方式**：角色隐瞒 + 反常行为
- **推进记录**：
  - 第2章：深夜电话事件
- **预计收束**：未知
- **收束方向**：李四在隐瞒什么？电话对象是谁？

## 疑似伏笔

### 反复出现的猫
- **出现章节**：第1、3、5章
- **描述**：一只黑猫在不同场景中出现
- **判断依据**：出现频率高于普通环境描写，可能是有意为之
- **建议**：请作者确认是否为有意设置的伏笔

## 已收束伏笔

### 张三的旧照片
- **埋设**：第1章（张三翻到一张模糊的合影）
- **收束**：第6章（揭示合影中的人是张三的亲生父亲）
- **方式**：通过对话揭示
```

## 执行步骤

1. 逐章通读，标记所有可能的伏笔点
2. 交叉比对：同一伏笔在不同章节的出现
3. 判断每个伏笔的状态（已埋/推进中/待收束/已收束）
4. 识别植入方式
5. 分离"明确伏笔"和"疑似伏笔"
6. 按模板格式输出
7. 附上"待确认问题"列表

---

**使用方式**：将此prompt和用户的文本一起发送给AI。建议提供全部已完成章节，以便准确追踪伏笔的推进。
