# 贡献指南

感谢你对 NovelForge 的关注！欢迎任何形式的贡献。

## 如何贡献

### 报告问题

- 使用 GitHub Issues 提交 bug 报告或功能建议
- 描述清楚问题的复现步骤
- 附上相关的错误信息或截图

### 提交代码

1. Fork 本仓库
2. 创建你的特性分支：`git checkout -b feature/你的特性名`
3. 提交你的修改：`git commit -m '添加某特性'`
4. 推送到分支：`git push origin feature/你的特性名`
5. 提交 Pull Request

### 代码规范

- JavaScript 文件使用 2 空格缩进
- Markdown 文件使用 UTF-8 编码
- 提交信息使用中文，简洁描述改动内容
- 模板文件中的占位符使用 `{{变量名}}` 格式

### 贡献方向

我们特别欢迎以下方面的贡献：

- **新的导入 prompt**：支持更多类型的文本分析
- **知识库模板优化**：更好的模板结构和引导注释
- **工具改进**：CLI 工具的功能增强
- **文档翻译**：英文文档、其他语言支持
- **场景范例**：通用的写作场景范例（不含具体作品内容）

## 行为准则

- 尊重每一位贡献者
- 保持友善和建设性的讨论
- 不提交任何具体作品的版权内容
