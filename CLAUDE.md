## 工作流程

### 创作新章节（6 阶段）

1. **章纲发散**：读完以下文件 → 生成 3 版章纲供人类选择
   - `00_核心上下文/` 全部文件（当前状态、最近章节梗概、伏笔追踪表、数值速查表、文风规则、写前护栏）
   - 涉及的人物档案（`01_人物档案/`）
2. **人类选择**：人类选定/修改章纲
3. **正文产出**：按选定章纲写完整章节（6000-10000 字），写前再读一遍：
   - 相关人物档案（校准语感）
   - `04_写作参考/` 中本章涉及的场景范例
   - `04_写作参考/微调意图学习库.md` → 应用人类已确认的偏好模式
4. **AI 自查**：正文写完后、交给人类前，读取以下文件逐条核验：
   - `04_写作参考/错题集_完整版.md` → 先看底部「场景类型索引」，定位本章最相关的 5-10 条，逐条对照正文
   - `04_写作参考/动作细节去重表.md` → 检查高频动作是否超出保留建议次数
   - `04_写作参考/微调意图学习库.md` → 对照隐性规则总结，检查是否违反已知偏好
   - 对照下方「终版自查清单」逐项核对
   - 能自行修复的问题直接修复，不确定的标注出来交给人类
   - 输出：修正后的正文 + 简要自查报告
5. **问题修正**：人类反馈 → 修订 → 如有新错误模式则记入错题集
6. **人类微调 + 学习库更新**：人类微调后，AI 分析所有修改 → 提取意图 → 更新 `微调意图学习库.md` → 自我纠正文档中的不合理处
7. **终版归档 + 知识库更新**：人类确认终版后，按下方清单更新知识库

### 章纲输出格式

每版章纲应包含以下结构：

```markdown
## 版本X：【一句话侧重点】

### 核心事件
- 事件1
- 事件2

### 关键场景
1. 场景（地点、人物、冲突点）
2. 场景（地点、人物、冲突点）

### 人物状态变化
- 角色A：从__到__
- 角色B：从__到__

### 伏笔操作
- 推进：伏笔名（来自伏笔追踪表）
- 新埋：伏笔名（附植入方式：体感/双线信息差/日常动作/对话带出）

### 情绪节奏与详略
- **开场**：情绪基调（如：压抑/松弛/紧张）
- **场景1**：↑/↓/→ 一句话情绪走向 【详写/略写】理由
- **场景2**：↑/↓/→ 一句话情绪走向 【详写/略写】理由
- **收束**：落在__情绪，方式：__（参考`场景范例_章末收束.md`）
- **整章弧线**：一句话（如"紧张→喘息→更深的不安"）

### 预计字数
约XXXX字
```

> **为什么需要「情绪节奏与详略」**：AI 倾向于给每个场景分配均等篇幅，导致节奏平坦。在章纲阶段就标注详略和情绪走向，能在动笔前解决结构层问题，避免写完再大改。详写场景通常不超过2个，其余应明确标注为略写。

> **伏笔植入前三问**（新埋伏笔时必答）：
>
> 1. **何时？** 当前场景的情绪节奏允许吗？
> 2. **谁的视角？** 当前角色有理由注意到这个细节吗？
> 3. **植入后能否立刻走？** 伏笔说完就回主线，不多停留解释。

### 终版自查清单

AI 自查阶段（阶段4）及人类确认终版前，逐项核对：

**设定层**
- [ ] 数值符合数值速查表？
- [ ] 机制符合世界观设定？
- [ ] 时间线与当前状态一致？人物位置合理？

**人物层**
- [ ] 对照人物档案和语言速查表，性格/说话方式/行为逻辑是否 OOC？
- [ ] 角色困境与前次是否不同？体现了渐进性成长？
- [ ] 角色用符合其知识水平的语言描述事物？

**结构层**
- [ ] 承接上一章结尾？事件动机充分？
- [ ] 三人以上平行段落是否均分了篇幅？→ 只详写1人，其余差异化略写
- [ ] 世界观信息是否滑入了课堂模式？→ 说话人必须有自己的情绪和立场

**文字层（叙事效率——能删则删，流畅优先）**
- [ ] 是否有不推进故事也不推进人物的环境描写或动作描写？→ 删
- [ ] 场景入场是否用了超过2句的空间布局描写？→ 压缩到一个锚点
- [ ] 同一情绪是否有 ≥2 个动作在表达？→ 只保留最精准的一个
- [ ] 章内是否存在频繁的单句成段（超过5处）？→ 合并普通叙述段落
- [ ] 同一信息在章内出现了几次？→ 首次完整，后续递减
- [ ] 是否有"他知道/他明白/他心里清楚"式心理总结句？→ 先试删
- [ ] 是否有"不是X。是Y。"式的AI味判断句？→ 改为自然叙述
- [ ] 章末最后一句删掉，全章意思仍完整？→ 完整则删

**伏笔层**
- [ ] 推进了哪些伏笔？新埋了哪些？有无矛盾？
- [ ] 新埋伏笔通过了上方「伏笔植入前三问」？

**章末**
- [ ] 结尾是否自然？好奇心来自情境本身还是叙述者硬塞？→ 后者删
- [ ] 如果已有自然张力，不要额外添加悬念钩子

- [ ] `00_核心上下文/伏笔追踪表_完整版.md` — 更新伏笔状态、新增伏笔、收束伏笔
- [ ] `00_核心上下文/数值速查表.md` — 如有新数值出现则补充
- [ ] `01_人物档案/` — 如人物状态有重大变化则更新对应档案
- [ ] `04_写作参考/错题集_完整版.md` — 如本轮修订中发现了新的错误模式则记录

- [ ] `04_写作参考/微调意图学习库.md` — **人类微调后必更新**：分析所有修改 → 提取意图 → 补充新案例 → 更新隐性规则

## 核心文风规则（速查）

> 此部分应根据 `知识库/00_核心上下文/文风规则_完整版.md` 定制

- **视角**：第三人称角色受限视角，只写对情节或情绪有用的
- **句式**：自然句长为常态（15-30字），短句（10字以内）仅用于真正的冲击点
- **情绪**：叙事推进本身就是情绪表达。动作藏情绪是选项不是义务
- **叙事效率**：场景入场一个锚点就够；过渡段能一句带过就不写三句
- **对话**：口语化，符合角色年龄和身份
- **标点规范**：正文中引号使用中文双引号""，禁用破折号——（用逗号、句号或换行替代）
