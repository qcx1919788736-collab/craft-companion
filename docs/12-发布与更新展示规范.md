# 发布与更新展示规范

本规范用于保证每次发布都“可见、可复现、可验证”。

## 最小发布产物

每个版本至少包含：
- `CHANGELOG.md` 对应版本条目
- 一份发布说明（建议放在 `docs/releases/`）
- README 中“最新更新”入口
- 升级后可执行的验收命令

## 推荐展示结构

1. 本次解决的问题
2. 3-5 个核心亮点
3. 升级命令
4. 升级前后对比
5. 兼容性说明

## 发布检查清单

- [ ] 版本号已更新（`package.json`）
- [ ] Changelog 已落版（含日期）
- [ ] README 版本徽章与文档链接已更新
- [ ] `bootstrap-entry` 与 `doctor` 可执行
- [ ] 至少完成一次本地跑通验证

## 建议的发布节奏

- 日常迭代：更新 `Unreleased`
- 对外发布：落版为正式版本（如 `1.2.0`）
- 同步发布说明并给出升级命令
