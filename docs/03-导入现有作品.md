# 导入现有作品

如果你已经有一些章节文本，本指南帮助你将它们导入到 NovelForge 知识库。

## 概述

导入过程通过 5 个专用 prompt 完成，每个 prompt 负责提取一类信息：

| 步骤 | Prompt | 提取内容 | 输出文件 |
|------|--------|---------|---------|
| 1 | 提取人物档案 | 角色信息 | `01_人物档案/` |
| 2 | 提取世界观设定 | 设定信息 | `02_世界观设定/` |
| 3 | 构建时间线 | 时间信息 | `03_故事进展/时间线.md` |
| 4 | 分析文风特征 | 写作风格 | `00_核心上下文/文风规则.md` |
| 5 | 识别伏笔线索 | 伏笔信息 | `00_核心上下文/伏笔追踪表.md` |

## 准备工作

### 文本要求

- 格式：纯文本或 Markdown
- 编码：UTF-8
- 建议至少 3 章 / 15000 字以上（文风分析需要足够样本）
- 章节之间有明确分隔

### 初始化项目

```bash
node tools/init.js
# 输入项目名称
cd 你的项目名
```

## 导入步骤

### 步骤 1：提取人物档案

**Prompt 文件**：`prompts/import/01-extract-characters.md`

**操作**：将 prompt 内容和你的全部文本一起发送给 AI

**AI 会做什么**：
- 识别所有出现的角色
- 分析角色的性格、说话方式、行为模式
- 梳理角色之间的关系
- 为每个主要角色生成档案

**输出**：每个主要角色一个 `.md` 文件

**人类审核要点**：
- AI 推测的性格是否准确？
- 角色关系是否遗漏？
- 说话方式的示例是否典型？

### 步骤 2：提取世界观设定

**Prompt 文件**：`prompts/import/02-extract-worldview.md`

**操作**：将 prompt 和文本发送给 AI

**AI 会做什么**：
- 识别核心机制（能力体系/科技/魔法等）
- 提取社会背景和地理信息
- 区分明确设定和推测设定
- 提取数值参数

**输出**：按主题分类的设定文档

**人类审核要点**：
- "推测设定"是否正确？
- 有没有遗漏的重要设定？
- 数值参数是否准确？

**注意**：现实题材作品可能不需要这一步，或只需要记录与现实不同的部分。

### 步骤 3：构建时间线

**Prompt 文件**：`prompts/import/03-extract-timeline.md`

**操作**：将 prompt 和文本发送给 AI

**AI 会做什么**：
- 提取所有时间标记
- 按时间顺序排列事件
- 识别时间跳跃和空白期
- 标注时间矛盾

**输出**：`03_故事进展/时间线_完整版.md`

**人类审核要点**：
- 时间推算是否合理？
- 有没有遗漏的重要事件？
- 时间矛盾如何解决？

### 步骤 4：分析文风特征

**Prompt 文件**：`prompts/import/04-analyze-style.md`

**这是最重要的一步。** 文风规则直接决定 AI 续写时能否"像你"。

**操作**：将 prompt 和尽可能多的文本发送给 AI

**AI 会做什么**：
- 量化分析句式特征（句长、段落结构）
- 分析情绪表达方式
- 分析对话风格
- 分析场景描写习惯
- 生成可执行的文风规则

**输出**：`00_核心上下文/文风规则_完整版.md`

**人类审核要点**：
- 规则是否反映了你的真实风格？
- 有没有你不认同的规则？
- 规则是否足够具体（可量化）？

**建议**：文风分析后，让 AI 用这些规则试写一段，看看效果。如果不像你，调整规则。

### 步骤 5：识别伏笔线索

**Prompt 文件**：`prompts/import/05-extract-threads.md`

**操作**：将 prompt 和全部文本发送给 AI

**AI 会做什么**：
- 识别所有伏笔和线索
- 判断伏笔状态（已埋/推进中/待收束/已收束）
- 识别植入方式
- 分离明确伏笔和疑似伏笔

**输出**：`00_核心上下文/伏笔追踪表_完整版.md`

**人类审核要点**：
- "疑似伏笔"中哪些是有意为之？
- 有没有遗漏的伏笔？
- 伏笔状态判断是否准确？

## 导入后的工作

### 1. 初始化当前状态

导入完成后，手动创建 `00_核心上下文/当前状态.md`：

```markdown
# 当前状态

## 最新进度
- **最新终版**：第X章
- **待创作**：第X+1章
- **故事时间点**：[从时间线提取]

## 主要角色状态
- [角色1]：[当前状态]
- [角色2]：[当前状态]

## 待推进事项
- [从伏笔追踪表提取]
```

### 2. 生成最近章节梗概

为最近 5 章写简要梗概，保存到 `00_核心上下文/最近章节梗概.md`。

### 3. 初始化错题集

创建空的 `04_写作参考/错题集_完整版.md`，随着创作积累错误模式。

### 4. 定制 CLAUDE.md

根据你的作品特点，修改 `CLAUDE.md` 中的：
- 人物语言速查表
- 核心红线
- 自查清单

## 常见问题

**Q: 文本量不够怎么办？**

A: 可以先导入，标注"样本量不足"。随着创作推进，逐步完善知识库。

**Q: 导入结果不准确怎么办？**

A: 导入结果是初稿，需要人类审核和修正。AI 的分析不可能 100% 准确。

**Q: 可以只导入部分步骤吗？**

A: 可以。但步骤 4（文风分析）强烈建议执行。

**Q: 导入后可以修改吗？**

A: 当然。知识库是活的文档，随时可以修改和完善。
