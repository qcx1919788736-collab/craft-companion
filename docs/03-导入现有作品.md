# 导入现有作品

如果你已经有一些章节文本，本指南帮助你把它们导入到 Craft Companion。

## 导入入口

请先创建项目，并在初始化时选择：

- **导入已有小说**

如果你是已有本地项目，且不想重建目录，可在项目根目录运行：

```bash
craft-companion bootstrap-entry
craft-companion doctor
```

该命令会补齐入口和基础骨架，不会覆盖你已有的正文章节文件。

然后进入你的项目目录，优先阅读：

- `START_HERE.md`

---

## 导入使用的 5 份模板

请按顺序使用：

| 步骤 | 模板文件 | 作用 |
|------|----------|------|
| 1 | `提示模板/导入已有小说/01-提取人物信息.md` | 提取人物档案、关系网络 |
| 2 | `提示模板/导入已有小说/02-提取世界观设定.md` | 整理世界观、机制、规则 |
| 3 | `提示模板/导入已有小说/03-构建时间线.md` | 重建故事时间线 |
| 4 | `提示模板/导入已有小说/04-分析文风特征.md` | 分析语言风格与文风规则 |
| 5 | `提示模板/导入已有小说/05-识别伏笔线索.md` | 梳理伏笔、线索、悬念 |

---

## 推荐流程

1. 先完成人物、世界观、时间线（1-3）
2. 再做文风分析（4）
3. 最后做伏笔识别（5）

> 文风分析通常最关键，它直接影响 AI 后续是否“像你”。

---

## 辅助命令

如果你想让 CLI 帮你直接打印这些模板，可以在项目目录中运行：

```bash
node tools/import-cli.js
```

它会按当前新版路径显示模板内容，不再使用旧的 `prompts/import/...`。

---

## 导入完成后建议检查

至少确认这些内容已经落地到知识库：

- 人物档案
- 世界观设定
- 时间线
- 文风规则
- 伏笔追踪表

然后再进入正式创作。

建议进入 `提示模板/通用流程/`，按“执行层自查 + 评估层复核”模式开始章节协作。
