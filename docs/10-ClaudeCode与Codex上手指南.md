# Claude Code 与 Codex 上手指南

本指南目标：让新用户把项目导入后，5 分钟内进入可执行状态。

## 推荐目录方式

优先使用 `node tools/init.js` 创建项目目录，再把“新建项目目录”作为工作目录导入 Claude Code 或 Codex。

这样做的好处：
- 项目目录内已包含 `CLAUDE.md`、`START_HERE.md`
- 已复制 `提示模板/` 与 `docs/`
- 不需要来回切父目录找模板

如果是已有项目，先在项目根目录运行：

```bash
craft-companion bootstrap-entry
```

## 首轮对话模板（直接可用）

把下面这句作为第一条消息：

```text
请先读取 CLAUDE.md 和 START_HERE.md，然后严格按 START_HERE.md 的第1步带我执行，不要提前写正文。
```

## 推荐工作节奏

1. 先完成初始化模板（从零开始或导入已有小说）
2. 再进入 5 阶段创作
3. 每章至少产出两份报告：执行层自查、评估层复核
4. 有争议问题用“原文 + 建议改写块”并排保留

## 常见错误

- 一上来直接让 AI 写正文
- 把章纲/初稿/自查混成一步
- 只做自查，不做评估层复核
- 争议问题直接改掉，导致后续返工

## 最低验收标准（每章）

- 有章纲选择记录
- 有初稿
- 有执行层自查报告
- 有评估层复核报告
- 终版前无未裁决争议
