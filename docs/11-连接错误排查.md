# 连接错误排查（含 TLS: bad record MAC）

## 典型报错

- `local error: tls: bad record MAC`

该错误通常来自网络链路或代理层，不是项目 Markdown 文件本身损坏。

## 快速判断

1. 同一会话里，文件读写功能是否正常
2. 报错是否出现在模型请求阶段，而非本地脚本执行阶段
3. 重试后是否偶发恢复

如果 1 正常、2 成立，基本可判断是连接层问题。

## 建议处理顺序

1. **先重试当前请求**（短时抖动最常见）
2. **切换网络**（如从 Wi-Fi 切手机热点）
3. **检查代理/VPN**：关闭后重试，或切换节点
4. **统一 TLS 终端**：避免多层代理串联（本地代理 + 系统代理 + 客户端代理）
5. **降低并发请求**：多开会话可能放大链路不稳定
6. **重启客户端**（Claude Code / Codex / Cherry Studio）

## 项目侧兜底建议

即使连接抖动，也要保证可恢复：
- 每阶段写检查点（见 `docs/08-检查点机制.md`）
- 中断恢复时先读 `当前状态.md` 和最新 checkpoint
- 有争议改动先写“建议改写块”，避免直接覆盖

## 何时需要进一步排查

满足以下任一项，建议检查系统与网络环境：
- 连续 10 分钟内高频出现同一 TLS 报错
- 同网络下多个客户端都报错
- 仅在特定代理节点触发

## 给用户的简短回复模板

```text
这是连接层 TLS 抖动，不是你项目文件损坏。先重试一次；若仍报错，切换网络或代理节点再试。你的本地文档和工作区内容是安全的，我们可以从当前状态继续。
```
