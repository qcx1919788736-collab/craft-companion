# 工作流详解

## 6阶段工作流

NovelForge 采用 6 阶段工作流，每个阶段都有明确的目标和质量控制。

```
阶段1: 章纲发散
   ↓
阶段2: 选择
   ↓
阶段3: 正文产出
   ↓
阶段4: AI自查
   ↓
阶段5: 人类修正
   ↓
阶段6: 归档
```

## 阶段1：章纲发散

**目标**：生成多个可能的章节方向

**输入**：
- 当前故事状态
- 下一步的可能性

**AI 的工作**：
- 基于当前状态，发散出 3-5 个可能的章节方向
- 每个方向简述核心事件和人物变化

**输出**：
- 3-5 个章纲选项

**质量控制**：
- 每个选项都要符合世界观和人物设定
- 要考虑伏笔的推进

## 阶段2：选择

**目标**：人类选择最满意的方向

**人类的工作**：
- 阅读 AI 生成的章纲选项
- 选择一个，或提出修改意见
- 可以混合多个选项

**输出**：
- 确定的章节方向

## 阶段3：正文产出

**目标**：根据选定的章纲，生成正文

**AI 的工作**：
- 严格遵循知识库中的设定和文风规则
- 按照章纲展开情节
- 注意人物一致性和伏笔推进

**输出**：
- 章节初稿

**质量控制**：
- 对照 `文风规则.md` 检查
- 对照 `人物档案` 检查角色一致性
- 对照 `伏笔追踪表` 检查伏笔推进

## 阶段4：AI自查

**目标**：AI 自己检查初稿的问题

**AI 的工作**：
- 对照 `错题集.md` 检查是否犯了已知错误
- 对照 `文风规则.md` 检查文风一致性
- 对照 `动作细节去重表.md` 检查是否重复
- 检查人物行为是否符合档案

**输出**：
- 自查报告（列出发现的问题）
- 修正后的版本

**质量控制**：
- 自查要诚实，不要掩盖问题
- 修正要彻底，不要敷衍

## 阶段5：人类修正

**目标**：人类最终审核和修改

**人类的工作**：
- 阅读 AI 的自查报告和修正版
- 进行最终修改
- 记录修改意图（重要！）

**输出**：
- 终稿
- 修改意图记录（用于更新知识库）

## 阶段6：归档

**目标**：更新知识库，为下一章做准备

**AI 的工作**：
- 更新 `当前状态.md`
- 更新 `最近章节梗概.md`
- 更新 `伏笔追踪表.md`
- 如果有新问题，添加到 `错题集.md`
- 如果有新的文风规律，添加到 `微调意图学习库.md`

**输出**：
- 更新后的知识库

## 工作流变体

### 快速模式（跳过发散）

如果你已经明确知道下一章要写什么：

```
章纲（人类提供） → 正文产出 → AI自查 → 人类修正 → 归档
```

### 探索模式（多轮发散）

如果你不确定方向：

```
章纲发散 → 选择 → 再发散 → 再选择 → ... → 正文产出
```

### 修订模式（已有初稿）

如果你已经有初稿，只需要优化：

```
初稿 → AI自查 → 人类修正 → 归档
```

## 实践建议

### 阶段1-2：不要急于确定

- 多生成几个选项，不要满足于第一个
- 考虑长期影响，不要只看当前章节
- 可以混合多个选项的优点

### 阶段3：信任知识库

- AI 应该严格遵循知识库，不要"创新"
- 如果发现知识库不够用，先暂停，补充知识库
- 不要在正文产出阶段修改设定

### 阶段4：诚实自查

- AI 要诚实指出问题，不要掩盖
- 对照 `错题集.md` 是最重要的
- 发现问题后要彻底修正，不要敷衍

### 阶段5：记录意图

- 修改时记录"为什么这样改"
- 这些意图会进入 `微调意图学习库.md`
- AI 会从中学习你的偏好

### 阶段6：及时归档

- 不要拖延，每章完成后立即归档
- 归档是为下一章做准备，不是负担
- 归档质量直接影响下一章的质量

## 常见问题

### Q: 每个阶段都必须执行吗？

A: 不是。根据实际情况灵活调整。但阶段4（AI自查）和阶段6（归档）强烈建议不要跳过。

### Q: 阶段3和阶段4可以合并吗？

A: 不建议。分开执行可以让AI更专注，自查更彻底。

### Q: 如果AI自查没发现问题怎么办？

A: 这是正常的。不是每章都有问题。但如果连续多章都"没问题"，可能是自查不够严格。

### Q: 归档需要多长时间？

A: 通常5-10分钟。主要是更新 `当前状态.md` 和 `最近章节梗概.md`。
