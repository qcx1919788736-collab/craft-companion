# 体验验收与一致性检查清单

目标：确保“用户能顺利上手、流程能稳定接力、质量检查可落地”。

## 场景A：新用户从零开始

1. 执行 `node tools/init.js`
2. 打开新项目目录
3. 发送首轮消息：
   - `请先读取 CLAUDE.md 和 AI入口_统一指令.md，然后带我完成第一步，不要提前写正文。`
4. 验收点：
   - AI 先分流，不直接写正文
   - 能进入 `提示模板/从零开始/01-定义核心概念.md`

## 场景B：已有项目补齐入口

1. 在项目根目录执行 `craft-companion bootstrap-entry`
2. 执行 `craft-companion doctor`
3. 验收点：
   - `START_HERE.md`、`AI入口_统一指令.md` 已存在
   - `当前状态.md` 为“进度接力”格式
   - `doctor` 无失败项

## 场景C：章节流程一致性

1. 触发“写下一章”
2. 验收点：
   - 先给章纲候选并停住等待选择
   - 进入自查时有“执行层 + 评估层”双层输出
   - 有争议项时保留“原文 + 建议改写块”

## 场景D：连接抖动恢复

1. 模拟中断或连接异常
2. 验收点：
   - 能根据 `当前状态.md` 和检查点继续
   - 若出现 `tls: bad record MAC`，能按 `docs/11-连接错误排查.md` 处理

## 最终判定

- 满足 A/B/C 为“可对外使用”
- 满足 A/B/C/D 为“可稳定协作”
