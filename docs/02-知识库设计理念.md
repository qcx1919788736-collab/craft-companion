# 知识库设计理念

Craft Companion 的核心是一个结构化知识库系统。它的目标不是“把所有资料都塞给 AI”，而是让 AI 在需要的时候读取正确的信息，减少跑偏。

## 为什么需要结构化知识库

长篇小说协作最容易出现的问题有：
- 人物写崩
- 世界观设定冲突
- 时间线混乱
- 伏笔遗忘
- 文风不稳定

结构化知识库的作用，就是把这些关键信息从“靠 AI 临时记忆”变成“可查、可更新、可约束”的外部资料。

## 核心原则

1. **唯一真相源**：设定以知识库为准
2. **分层存放**：不同类型信息放在不同目录
3. **逐步更新**：每章终版后再更新知识库
4. **先初始化再创作**：不要跳过从零开始/导入已有小说的准备阶段

## 目录结构

- `00_核心上下文/`
- `01_人物档案/`
- `02_世界观设定/`
- `03_故事进展/`
- `04_写作参考/`

## 使用建议

- 从零开始用户：先按 `提示模板/从零开始/` 初始化
- 导入用户：先按 `提示模板/导入已有小说/` 完成知识提取
- 正式创作时，优先读取 `CLAUDE.md`、`START_HERE.md` 和当前状态文件
