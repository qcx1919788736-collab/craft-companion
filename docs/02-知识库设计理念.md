# 知识库设计理念

NovelForge 的核心是一个结构化的知识库系统。理解其设计理念，能帮助你更好地使用和定制这个框架。

## 核心问题

AI 协作创作面临的核心问题：

1. **一致性问题**：AI 容易忘记之前的设定，导致前后矛盾
2. **风格漂移**：AI 的写作风格会逐渐偏离作者的风格
3. **重复错误**：AI 会反复犯同样的错误
4. **信息过载**：把所有内容都塞给 AI，反而降低质量

## 设计原则

### 1. 知识库是唯一真相源

所有设定、规则、状态都记录在知识库中。AI 和人类都以知识库为准，不依赖记忆。

**为什么**：AI 的上下文窗口有限，记忆不可靠。外部化的知识库是唯一可靠的真相源。

### 2. 分层加载，按需读取

知识库分 5 层，不是每次都全部加载：

- `00_核心上下文/` - 每次必读
- `01_人物档案/` - 涉及该人物时读
- `02_世界观设定/` - 涉及设定时读
- `03_故事进展/` - 需要回溯时读
- `04_写作参考/` - 特定阶段读

**为什么**：减少信息噪音，让 AI 专注于当前任务。

### 3. 规则可执行，不要模糊

文风规则要量化：
- ❌ "适当使用短句"
- ✅ "短句（10字以内）每章不超过5处"

**为什么**：模糊规则等于没有规则。AI 需要明确的边界。

### 4. 错误驱动学习

`错题集.md` 记录 AI 犯过的错误，每次写作前对照检查。

**为什么**：AI 会重复犯错，但如果明确告诉它"这是错的"，它能避免。

### 5. 人类意图学习

`微调意图学习库.md` 记录人类的修改意图，AI 从中学习偏好。

**为什么**：人类的修改包含隐性规则，提取出来可以减少未来的修改。

## 知识库结构详解

### 00_核心上下文

**作用**：每次创作必读的核心信息

**关键文件**：
- `当前状态.md` - 故事当前进度、人物位置、待推进事项
- `文风规则_完整版.md` - 写作风格的明确规则
- `伏笔追踪表_完整版.md` - 所有伏笔的状态追踪
- `数值速查表.md` - 数值一致性保证
- `最近章节梗概.md` - 最近 5 章的简要回顾

**设计考量**：
- 信息密度高，避免冗余
- 动态更新，每章完成后必须更新
- 5 章窗口：太少缺少上下文，太多信息过载

### 01_人物档案

**作用**：角色的完整设定和状态追踪

**格式**：每个主要角色一个文件

**关键内容**：
- 性格核心（3 个关键词）
- 说话方式（带示例）
- 行为逻辑
- 关系网络
- 状态变化记录

**设计考量**：
- 性格要可执行：不是"复杂"，而是"谨慎+被动+压抑"
- 语言速查表：AI 最容易写错对话，需要明确示例
- 动态更新：人物会成长，档案要跟着变

### 02_世界观设定

**作用**：世界规则、背景、术语

**适用场景**：
- 奇幻/科幻作品的能力体系
- 特殊历史背景
- 虚构的社会结构

**设计考量**：
- 区分"明确设定"和"角色认知"
- 记录数值参数，同步到数值速查表
- 现实题材只记录与现实不同的部分

### 03_故事进展

**作用**：时间线和历史章节回顾

**关键文件**：
- `时间线_完整版.md` - 事件的时间顺序
- `历史章节梗概.md` - 早期章节的简要回顾

**设计考量**：
- 时间线防止时间矛盾
- 历史梗概压缩信息，避免重读旧章节
- 最近章节放在 `00_核心上下文/`，历史章节放这里

### 04_写作参考

**作用**：质量控制和风格学习

**关键文件**：
- `错题集_完整版.md` - AI 犯过的错误模式
- `场景范例_*.md` - 优质场景示例
- `微调意图学习库.md` - 人类修改意图提取
- `动作细节去重表.md` - 避免重复描写

**设计考量**：
- 错题集是最重要的质量保证
- 场景范例比抽象规则更有效
- 意图学习让 AI 理解"为什么这样改"

## 工作流与知识库的关系

6 阶段工作流对应知识库的不同使用方式：

| 阶段 | 读取 | 写入 |
|------|------|------|
| 1. 章纲发散 | 00_核心上下文 + 人物档案 | 章纲草案 |
| 2. 选择 | - | 选定章纲 |
| 3. 正文产出 | 文风规则 + 场景范例 + 微调意图 | 初稿 |
| 4. AI 自查 | 错题集 + 去重表 | 自查报告 + 修正稿 |
| 5. 人类修正 | - | 终稿 |
| 6. 归档 | - | 更新知识库 |

**关键点**：
- 阶段 3 写作前再读一遍文风规则，校准语感
- 阶段 4 自查是质量保证的关键
- 阶段 6 归档决定下一章的质量

## 为什么是 Markdown

知识库全部使用 Markdown 格式：

**优点**：
- 纯文本，易于版本控制
- 人类可读，AI 可解析
- 跨平台，无需特殊工具
- 可以用任何编辑器修改

**缺点**：
- 没有数据库的查询能力
- 没有强制的结构验证

**权衡**：对于小说创作，可读性和灵活性比查询能力更重要。

## 扩展性

这个知识库结构可以适应不同类型的作品：

**短篇小说**：
- 简化 `03_故事进展/`
- 可能不需要 `伏笔追踪表`

**长篇系列**：
- 增加 `02_世界观设定/` 的细节
- `03_故事进展/` 可能需要分卷

**多视角作品**：
- `01_人物档案/` 为每个视角角色建档
- `文风规则` 可能需要区分不同视角的风格

**非虚构写作**：
- `02_世界观设定/` 改为事实核查清单
- `伏笔追踪表` 改为论点追踪
