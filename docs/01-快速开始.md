# 快速开始

本指南帮助你在 5 分钟内开始使用 NovelForge。

## 安装

```bash
git clone https://github.com/your-username/novelforge.git
cd novelforge
npm install  # 如果使用 CLI 工具
```

## 两种使用方式

### 方式一：导入现有作品

如果你已经有一些章节文本，可以通过导入工具快速建立知识库。

**步骤：**

1. **初始化项目**
   ```bash
   node tools/init.js
   # 输入项目名称，如：我的小说
   cd 我的小说
   ```

2. **运行导入工具**
   ```bash
   node ../tools/import-cli.js
   # 选择要执行的导入步骤
   ```

3. **按提示将 prompt 和你的文本发送给 AI**

   建议顺序：
   - 步骤 1：提取人物档案
   - 步骤 2：提取世界观设定
   - 步骤 3：构建时间线
   - 步骤 4：分析文风特征（最重要！）
   - 步骤 5：识别伏笔线索

4. **AI 会生成对应的知识库文件，保存到对应目录**

5. **开始创作新章节**

   将 `CLAUDE.md` 和知识库一起提供给 AI，按 6 阶段工作流创作。

### 方式二：从零开始

如果你还没有任何文本，从零构思一部作品。

**步骤：**

1. **初始化项目**
   ```bash
   node tools/init.js
   cd 我的小说
   ```

2. **使用引导 prompt 逐步构建**

   按顺序将以下 prompt 发送给 AI：

   - `prompts/bootstrap/01-define-core-concept.md` - 定义核心概念
   - `prompts/bootstrap/02-build-protagonist.md` - 构建主角
   - `prompts/bootstrap/03-design-structure.md` - 设计故事结构
   - `prompts/bootstrap/04-establish-style.md` - 建立文风
   - `prompts/bootstrap/05-ready-to-write.md` - 准备开始创作

3. **AI 会引导你完成知识库的初始化**

4. **开始写第一章**

## 创作工作流

知识库准备好后，按照 6 阶段工作流创作每一章：

```
1. 章纲发散 → AI 生成 3 个章纲选项
2. 选择     → 你选择一个方向
3. 正文产出 → AI 写完整章节
4. AI 自查  → AI 对照错题集和文风规则检查
5. 人类修正 → 你审阅并修改
6. 归档     → 更新知识库，为下一章做准备
```

**详细工作流说明**：见 `docs/04-工作流详解.md`

## 使用 CLI 工具（可选）

```bash
# 创建新章节工作文件
node tools/novel-cli.js new-chapter 1

# 归档已完成章节
node tools/novel-cli.js archive 1

# 检查知识库一致性
node tools/novel-cli.js check

# 显示知识库更新清单
node tools/novel-cli.js update-kb 1
```

## 核心文件说明

创建项目后，你会看到以下结构：

```
我的小说/
├── CLAUDE.md              # AI 协作指令（必读）
├── 知识库/
│   ├── 00_核心上下文/     # 当前状态、文风规则、伏笔追踪
│   ├── 01_人物档案/       # 角色设定
│   ├── 02_世界观设定/     # 世界观、设定
│   ├── 03_故事进展/       # 时间线、章节梗概
│   └── 04_写作参考/       # 错题集、场景范例
├── 工作区/                # 章节创作文件
└── _归档/                 # 已完成章节归档
```

**最重要的文件**：
- `CLAUDE.md` - AI 的工作手册，每次创作都要提供
- `知识库/00_核心上下文/文风规则_完整版.md` - 决定 AI 能否"像你"写作
- `知识库/00_核心上下文/当前状态.md` - 故事当前进度

## 常见问题

**Q: 必须使用 CLI 工具吗？**

A: 不是。CLI 工具只是辅助，核心是知识库 + CLAUDE.md + AI 对话。你可以完全手动管理文件。

**Q: 可以用其他 AI 模型吗？**

A: 理论上可以，但系统是针对 Claude 优化的。其他模型可能需要调整 prompt。

**Q: 知识库文件可以自己编辑吗？**

A: 完全可以！知识库是纯 Markdown，随时可以手动编辑。AI 只是帮你维护，不是强制的。

**Q: 导入工具会修改我的原始文件吗？**

A: 不会。导入工具只读取你的文本，生成新的知识库文件，不会修改原文件。
